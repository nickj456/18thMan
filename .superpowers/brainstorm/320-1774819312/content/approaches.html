Three Approaches for 18th Man

Pick the architecture that fits your timeline and goals

A

Next.js + Supabase ⭐ Recommended

Supabase handles auth, Postgres, real-time chat, and file storage in one platform. AI SDK for the coaching assistant. Fastest path to a working app with real-time features.

Pros

  • One platform for auth + DB + real-time + storage
  • Built-in Row Level Security for permissions
  • Real-time subscriptions = live chat out of the box
  • Large community, great docs

Cons

  • Vendor lock-in on Supabase
  • Less fine-grained auth control than Clerk
B

Next.js + Neon + Clerk + Pusher

Best-in-class for each concern. Clerk for auth (excellent UX + roles), Neon Postgres for data, Pusher for real-time messaging, Vercel Blob for uploads. More setup, more control.

Pros

  • Clerk has the best auth UX + RBAC out of the box
  • Each service is best-in-class
  • No single vendor dependency

Cons

  • More services to manage and bill separately
  • More initial configuration
C

Next.js + Convex

Convex is a reactive database purpose-built for real-time apps. Handles auth, data, and live subscriptions in one TypeScript-native platform. Excellent for chat-heavy apps.

Pros

  • Real-time is a first-class citizen
  • TypeScript throughout — no SQL needed
  • Built-in optimistic updates

Cons

  • Newer, smaller ecosystem
  • Less familiar than Postgres
  • Can get expensive at scale
